# Content Hub Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Transform mylottostats.com from a thin data tool into an editorial-grade lottery statistics platform for AdSense approval and traffic growth.

**Architecture:** Upgrade the blog generation pipeline to a category-based multi-pass system, enrich all thin pages with computed editorial prose, add E-E-A-T signals, and fix existing content quality bugs. All changes are SSG-compatible — no new serverless routes.

**Tech Stack:** Next.js 16 (App Router, SSG), TypeScript, Claude Sonnet API (blog generation), fs-based data loading

**Spec:** `docs/superpowers/specs/2026-03-25-content-hub-design.md`

**Note:** This project has no test framework configured. Verification is via `npm run build` (SSG build succeeds) and `npm run lint`. Each task ends with a build check.

**Note on HTML rendering:** The codebase already uses `dangerouslySetInnerHTML` for blog post content (see `src/app/blog/[slug]/page.tsx:90-100`). All HTML content is generated by our own scripts (Claude API output or static data files) — never from user input. This is an established pattern in the codebase and poses no XSS risk since there is no user-submitted HTML.

Add to `src/lib/data/fetcher.ts` after the `loadLotteryData` function:

```typescript
/**
 * Returns the total number of draws across all lottery data files.
 */
export function getTotalDrawCount(): number {
  const dataDir = path.join(process.cwd(), 'src', 'data');
  let total = 0;
  const files = fs.readdirSync(dataDir).filter(f => f.endsWith('.json'));
  for (const file of files) {
    try {
      const data: LotteryData = JSON.parse(fs.readFileSync(path.join(dataDir, file), 'utf-8'));
      total += data.draws.length;
    } catch { /* skip non-lottery JSON */ }
  }
  return total;
}
```
